Explanation

This is the A* path finding algorithm. Every time you press a key the algorithm will make a step towards finding the shortest path from A to B. You can use the mouse to add and remove walls.

The A* algorithm uses 3 values to determine what the next step towards the destination should be.

H (top left)
H is a value that describes the distance from a node to the goal.

G (top right)
G is a value that describes the distance that has to be traveled to get from the starting node to this node.

F (middle) G + H
F is the value that the algorithm uses to determine what the most promising node is to investigate next on it's journey towards finding the shortest path.

Parent (bottom middle)
In the A* algorithm each node keeps a set of coordinates to point to it's parent. These coordinates will ultimately be used make a chain (marked in blue when shortest path is found) from the goal back to the start.
